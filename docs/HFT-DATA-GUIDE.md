# A 股 1m K线数据指南

> 研究员设计因子前必读

## 数据源

- **Library**: `ashare@live@stock@kline@1m`
- **存储**: ArcticDB (S3)，per_symbol 布局
- **覆盖**: 5191 个 A 股 symbol
- **时间范围**: 2020-01-02 起
- **每日 bar 数**: 240 根

## 可用字段

| 字段 | 类型 | 可靠性 | 说明 |
|------|------|--------|------|
| `open` | float64 | 高 | 后复权开盘价 |
| `high` | float64 | 高 | 后复权最高价 |
| `low` | float64 | 高 | 后复权最低价 |
| `close` | float64 | 高 | 后复权收盘价 |
| `volume` | float64 | 高 | 成交量（股） |
| `amount` | float64 | 高 | 成交额（元） |
| `origin_open` | float64 | 高 | 未复权开盘价 |
| `origin_high` | float64 | 高 | 未复权最高价 |
| `origin_low` | float64 | 高 | 未复权最低价 |
| `origin_close` | float64 | 高 | 未复权收盘价 |

## 交易时间

| 时段 | 时间 | bar 数 | 说明 |
|------|------|--------|------|
| 上午 | 09:30-11:30 | 120 | 连续竞价 |
| 午休 | 11:30-13:00 | 0 | 无交易 |
| 下午 | 13:00-14:57 | 117 | 连续竞价 |
| 收盘集合竞价 | 14:57-15:00 | 3 | **特殊**：不是连续撮合 |

### 收盘集合竞价注意事项

14:57-15:00 的 3 根 bar 是收盘集合竞价阶段：
- 14:57 起停止连续撮合，只接受申报不成交
- 15:00 一次性以集合竞价方式确定收盘价
- 这 3 根 bar 的 OHLCV 数据特征与连续竞价完全不同（成交集中、价格跳变）
- **建议**：全天窗口使用 `("09:30", "11:30"), ("13:00", "14:57")` 排除收盘集合竞价

## 已知问题

### 1. 缺失 bar
实盘 1m 数据可能缺少 1-2 根 bar。`expected_bars` 设为窗口理论 bar 数的 2/3，低于此阈值的交易日跳过。

### 2. 日线数据与 1m 数据不一致
近期日期的 `ashare@live@stock@kline@1d` 的 high/low 可能仅反映上午行情（日线数据更新时序问题）。下午窗口的 twap/vwap 与日线 high/low 对比时可能出现越界。**这不是 1m 数据的问题。**

### 3. 停牌股
停牌期间无 1m bar 数据。framework 会跳过（NaN）。

### 4. 复权
- 因子计算**必须使用后复权**（hfq）字段：`open, high, low, close`
- `origin_*` 字段仅用于涨跌停判断等需要原始价格的场景
- 涉及绝对价格比较时，两侧必须在同一价格空间

## Numba 约束

所有 formula 必须 `@njit` 兼容：
- 纯 numpy 操作，不能用 pandas
- 显式 for 循环处理 NaN（不能用 np.nanmean 以外的高级函数）
- 返回 `np.array(dtype=np.float64)`
- 参考 `research/basic_rawdata/pv_stats/register_pv_stats_0930_1030.py`
